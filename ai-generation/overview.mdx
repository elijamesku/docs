## What is AI Code Generation?

Kyrna's AI code generation transforms **natural language descriptions** into **production-ready Terraform code**. Instead of manually writing hundreds of lines of HCL, you describe what you want in plain English, and Kyrna's AI does the rest.

## Example Generations

### Simple: Web Server

**Prompt:**
```text
Create a simple web server on AWS
```

**Generated:**
```hcl
resource "aws_instance" "web" {
  ami           = "ami-0c55b159cbfafe1f0"  # Ubuntu 22.04
  instance_type = "t3.micro"
  
  vpc_security_group_ids = [aws_security_group.web.id]
  
  user_data = <<-EOF
              #!/bin/bash
              apt-get update
              apt-get install -y nginx
              systemctl start nginx
              EOF
  
  tags = {
    Name = "web-server"
    ManagedBy = "Kyrna"
  }
}

resource "aws_security_group" "web" {
  name        = "web-sg"
  description = "Allow HTTP and SSH"
  
  ingress {
    from_port   = 80
    to_port     = 80
    protocol    = "tcp"
    cidr_blocks = ["0.0.0.0/0"]
  }
  
  ingress {
    from_port   = 22
    to_port     = 22
    protocol    = "tcp"
    cidr_blocks = ["0.0.0.0/0"]  # Should be restricted in production
  }
  
  egress {
    from_port   = 0
    to_port     = 0
    protocol    = "-1"
    cidr_blocks = ["0.0.0.0/0"]
  }
}

output "web_server_ip" {
  value = aws_instance.web.public_ip
}
```

**Cost:** ~$8/month

## Best Practices for Prompts

<Card title="Learn More" icon="lightbulb" href="/ai-generation/writing-prompts">
  See our complete guide on writing effective AI prompts →
</Card>

### Do's ✅

- Be specific about requirements
- Mention cloud provider explicitly
- Include environment (dev/prod)
- Specify constraints (budget, region)
- List integrations needed

### Don'ts ❌

- Don't be overly vague
- Don't assume implicit requirements
- Don't skip security requirements
- Don't forget to mention databases
- Don't ignore cost constraints

## Limitations & Considerations

<Warning>
  While Kyrna's AI is powerful, always review generated code before deploying to production.
</Warning>

### Current Limitations

- **Complex networking**: Very custom VPC designs may need manual adjustment
- **Legacy systems**: Older resource types may have limited support
- **Compliance**: Some industry-specific compliance needs manual verification
- **Third-party integrations**: Non-standard integrations may need custom code

### Best Practices

1. **Start simple**: Generate basic structure, then refine
2. **Review security**: Always review security group rules and IAM policies
3. **Test in dev**: Deploy to dev environment first
4. **Version control**: Commit generated code to Git
5. **Iterate**: Use follow-up prompts to refine
